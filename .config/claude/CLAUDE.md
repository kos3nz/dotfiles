## Rules

- In all interactions and commit messages, be extremely concise and sacrifice grammar for the sake of concision.

## PR Comments

PR TODO comments use checkbox format: `- [ ] description`

## Git

When creating branches, prefix them with kos3nz/ to indicate they came from me.

## Plan Mode

- Structure as phases/steps with checkboxes; mark when completed
- Break large phases into nested sub-steps
- Steps = small, atomic, executable dev tasks (not vague goals)
- After each step: stop and present changes for user review before proceeding
- Never batch multiple implementation steps without review checkpoint
- Append unresolved questions at end if any
