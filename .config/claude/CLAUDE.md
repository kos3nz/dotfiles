## Rules

- In all interactions and commit messages, be extremely concise and sacrifice grammar for the sake of concision.

## PR Comments

<pr-comment-rule>

When I say to add a comment to a PR with a TODO on it, use 'checkbox' markdown format to add the TODO. For instance:

<example>
- [ ] A description of the todo goes here
</example>

</pr-comment-rule>

## Git

When creating branches, prefix them with kos3nz/ to indicate they came from me.

## Plan Mode

- Make the plan extremely concise. Sacrifice grammar for concision.
- Structure the plan using phases and steps.
- Always include a checkbox for every phase and step.
- Mark the checkbox when a step is completed.
- If a phase covers a large scope, add nested phases or steps to break it down.
- Prefer short, actionable steps.
- Steps must be executable development tasks (not vague goals).
- At the end of the plan, include a list of unresolved questions (if any).
